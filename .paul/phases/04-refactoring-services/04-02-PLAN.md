  <name>Task 1: Session data trimming et tracking zombie dans command_handlers.py</name>
  <files>services/command_handlers.py</files>
  <action>
    **[S-05] Session data trimming:**
    - Dans TrackingHandler.update(), avant d'assigner current_status['session_data']:
      - Récupérer session_data = self.session.get_session_data()
      - Si session_data contient une liste 'corrections' (ou similaire), la tronquer aux 100 dernières entrées
      - Cela évite que le dict sérialisé en JSON à 20Hz grossisse indéfiniment

    **[S-11] Tracking zombie detection:**
    - Ajouter un attribut self._consecutive_errors = 0 dans __init__
    - Définir MAX_CONSECUTIVE_ERRORS = 5 comme constante de classe
    - Dans update(), dans le bloc except:
      - Incrémenter self._consecutive_errors
      - Logger avec logger.error(..., exc_info=True) pour avoir le traceback
      - Si self._consecutive_errors >= MAX_CONSECUTIVE_ERRORS:
        - Logger "Tracking stoppé après {N} erreurs consécutives"
        - Appeler self.stop(current_status)
        - Mettre current_status['status'] = 'error'
        - Mettre current_status['error'] = "Tracking stoppé: erreurs consécutives"
        - Mettre current_status['error_timestamp'] = time.time()
    - Dans update(), dans le bloc try (après succès):
      - Remettre self._consecutive_errors = 0

    **Éviter:**
    - Ne PAS changer la logique de start() ou stop()
    - Ne PAS modifier la structure de session_data (juste tronquer)
  </action>
