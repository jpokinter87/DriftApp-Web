<objective>
## Goal
Corriger le bug de perte de suivi au retournement méridien, causé par le watchdog systemd qui tue le motor_service pendant les longues rotations bloquantes, plus 3 bugs secondaires dans le code de correction tracking.

## Purpose
Lors d'un transit méridien, la coupole doit pivoter de 100-180° (ex: NGC 5033 de 246° à 112°). Cette rotation dure ~100 secondes en mode CONTINUOUS, mais le watchdog systemd (30s) tue le motor_service à 30s car la rotation bloquante empêche l'envoi de heartbeats. Le service redémarre frais, sans session de suivi → l'UI affiche "pas de cible" et l'utilisateur doit relancer manuellement.

**Preuve terrain** : Logs LBN 166.16+04.52 montrent 10 restarts successifs du motor_service (chacun avançant ~17° avant d'être tué), confirmant le kill watchdog.

## Output
- Watchdog sur thread dédié (heartbeat même pendant rotation bloquante)
- Bug `session.start()` tuple unpacking corrigé dans TrackingHandler
- position_relative normalisée dans le chemin sans feedback
- Re-sync encodeur après grands mouvements
- Détection et logging explicite des transits méridien
- Suite de tests dédiée
</objective>
