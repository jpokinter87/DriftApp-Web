<objective>
## Goal
Réaliser une revue de code exhaustive des modules Core : config, hardware (moteur, feedback, encodeur), et utilitaires. Identifier failles, bugs latents, code mort, simplifications possibles et dette technique.

## Purpose
Établir un diagnostic précis de la qualité du code Core avant tout refactoring. Ces modules sont critiques car ils contrôlent directement le hardware (GPIO, SPI, moteur pas-à-pas).

## Output
Rapport de review `.paul/phases/01-code-review-audit/01-01-REVIEW.md` avec findings catégorisés par sévérité.
</objective>
