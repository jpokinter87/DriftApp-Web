<objective>
## Goal
Créer un programme de tests terrain qui simule le comportement RÉEL de la monture équatoriale : un objet suivi pendant plusieurs heures, traversant le méridien, avec le flip de monture et les corrections de coupole correspondantes. Basé sur les vrais incidents NGC 5033 et LBN 166.16+04.52.

## Purpose
Tester de jour (sans télescope) la chaîne complète : coordonnées RA/DEC → calcul az/alt heure par heure → position coupole abaque → mode adaptatif → corrections. Le script reproduit les conditions exactes des incidents terrain pour valider les corrections des Phases 2 et 3.5.

## Output
- `scripts/test_terrain.py` : simulation temporelle d'objets réels sur la monture
- `tests/test_terrain_scenarios.py` : tests pytest des scénarios critiques
</objective>

  <name>Task 1: Script test_terrain.py — simulation monture</name>
  <files>scripts/test_terrain.py</files>
  <action>
    Créer `scripts/test_terrain.py` — simulation temporelle de la monture équatoriale.

    **Principe :**
    Le script simule ce que fait le tracking en production : pour un objet donné (RA/DEC), il calcule sa position dans le ciel toutes les N minutes et en déduit les corrections de coupole nécessaires. Il détecte le transit méridien via le changement de signe de l'angle horaire.

    **Architecture :**
    ```python
    #!/usr/bin/env python3
    """Simulation monture — tests terrain de jour."""
    import sys
    sys.path.insert(0, str(Path(__file__).parent.parent))
    ```

    **Objets à simuler (vrais coordonnées J2000) :**
    1. **NGC 5033** — RA=13h13m27s (198.36°), DEC=+36°35'38" (36.59°)
       - Objet de l'incident du 16/03 à 2h20
       - Transit méridien prévu, delta ~134°
    2. **LBN 166.16+04.52** — Utiliser les coordonnées du log (RA=81.89°, DEC=42.97°)
       - Objet du 2ème incident, proche du zénith au transit
    3. **M42** (Orion) — RA=83.82°, DEC=-5.39°
       - Objet standard, altitude moyenne, pas de flip critique

    **Pour chaque objet, le script doit :**
    1. Déterminer l'heure du transit méridien (quand angle_horaire ≈ 0)
    2. Simuler la trajectoire de transit-3h à transit+3h, pas de 10 min
    3. À chaque pas :
       a. Calculer az/alt via AstronomicalCalculations.calculer_coords_horizontales()
       b. Calculer angle horaire via calculer_angle_horaire()
       c. Calculer position coupole via AbaqueManager.get_dome_position()
       d. Calculer delta via AdaptiveTrackingManager.verify_shortest_path()
       e. Évaluer le mode via evaluate_tracking_zone()
    4. Détecter le transit méridien (angle_horaire change de signe)
    5. Au transit : vérifier que le delta correspond à un grand mouvement

    **Format de sortie :**
    ```
    ═══ SIMULATION MONTURE — NGC 5033 ═══
    Coordonnées J2000 : RA=198.36° DEC=+36.59°
    Site : Observatoire Ubik (44.15°N, 5.23°E)
    Transit méridien estimé : 02:18 UTC+1

    Heure     | HA      | Az     | Alt    | Coupole | Delta   | Mode
    ----------+---------+--------+--------+---------+---------+---------
    23:18     | -3h00   | 45.2   | 32.1   | 43.8    | +1.2    | normal
    23:28     | -2h50   | 47.5   | 33.8   | 46.1    | +1.1    | normal
    ...
    02:18     | +0h00   | 180.0  | 82.5   | 179.3   | +134.2  | continuous  ★ TRANSIT
    ...
    05:18     | +3h00   | 315.2  | 32.1   | 313.8   | +1.0    | normal

    ═══ RÉSULTAT ═══
    Transit détecté : 02:18 (HA=+0h02)
    Delta au transit : +134.2° → CONTINUOUS ✓
    Altitude au transit : 82.5° → CRITICAL zone ✓
    Cohérence abaque : toutes positions dans [0, 360] ✓
    PASS
    ```

    **Résumé final :**
    ```
    ═══ RÉSUMÉ ═══
    NGC 5033    : PASS (transit 02:18, delta=+134.2°)
    LBN 166.16  : PASS (transit 18:26, delta=+170.1°, alt=86.7°)
    M42         : PASS (pas de grand delta, mode normal)
    Total : 3/3 PASS
    ```

    **Vérifications automatiques par scénario :**
    - Transit détecté ± 30 min de l'heure attendue
    - Delta au transit > 30° pour NGC 5033 et LBN (grands mouvements)
    - Mode CONTINUOUS au moment du grand delta
    - Toutes les positions abaque dans [0, 360]
    - Pas de NaN ni d'erreur de calcul

    **Date de simulation :** Utiliser la date des incidents (2026-03-16) pour correspondre aux logs terrain. La date peut être passée en argument optionnel.

    Le script doit fonctionner SANS matériel (pas de GPIO, pas d'encodeur, pas de moteur). Il utilise uniquement les modules de calcul.
